# Redis セッション実装まとめ（Hono + Redis）

## 概要
今回の実装では、Hono アプリに Redis を組み合わせて **セッション認証機構** を構築した。
Docker Compose で Redis を起動し、TypeScript 側から Redis をセッションストアとして利用した。

---

# 実装内容

## 1. Redis を Docker で起動
Redis をローカルで利用するために Docker Compose を使用。

- `redis:7` イメージを利用
- ポート `6379` を公開
- `redis-data:/data` で永続化
- `appendonly yes` による永続化設定

### ポイント
- `/data` はコンテナ内のパス
- 実データは Docker volume に保存される

構造は以下の通り：

- イメージ: Redis の設計図（immutable）
- コンテナ: 実行環境
- ボリューム: データ保存領域（永続化）

---

## 2. Redis クライアント接続（Node.js / TypeScript）

Redis をセッションストアとして利用するため、アプリ起動時に接続。

### ログイン処理
- 認証成功
- セッションID生成
- Redis に保存

session:<sessionId> → user情報

- Cookie にセッションIDをセット

### 認証チェック（middleware）
- Cookie から sessionId を取得
- Redis からセッション取得
- 存在すれば認証OK

### ログアウト
- Redis のセッション削除
- Cookie 削除

---

## 3. セッションストアとして Redis を採用

理由：

- 高速な読み書き
- TTL による自動期限切れ
- 一時データに最適
- 認証・キャッシュ用途に適している

---

# 発生した問題

## 問題1: Redis エラー

The client is closed


状況：

- Redis サーバーは起動している
- しかし Node 側でエラー発生

ログ：


isOpen: true
isReady: false


---

## 問題2: ログイン後にセッション切れ

- ログインは成功する
- しかし次のリクエストで未認証になる

---

# 原因

## 原因1: Redis クライアントの状態管理

問題点：

- `connect()` 完了前にコマンド実行
- `quit()` 後に再利用
- client の使い回し不備

重要な理解：

- `isOpen` → ソケットが開いているだけ
- `isReady` → コマンド送信可能状態

---

## 原因2: セッションの受け渡し不整合

考えられる原因：

- Cookie が送られていない
- `credentials: "include"` が不足
- Cookie 設定不備（path / sameSite / secure）
- Cookie 名の不一致
- Redis キー名の不一致
- TTL の問題
- logout 時の誤削除

---

# 改善方法

## 改善1: Redis client を1つに統一

```ts
const redis = createClient()
await redis.connect()
```

アプリ起動時に1回だけ接続

使い回す

## 改善2: connect 完了後に使用
```ts
await redis.connect()
```
isReady を確認

接続完了前に操作しない

## 改善3: logout で client を閉じない

NG：
```ts
await redis.quit()
```

OK：
```ts
await redis.del(`session:${sessionId}`)
```

## 改善4: Cookie / fetch 設定の統一
フロント
```ts
fetch(url, {
  credentials: "include"
})
```
バックエンド
```ts
setCookie(c, "session_id", sessionId, {
  path: "/",
  httpOnly: true,
  sameSite: "Lax",
  secure: false,
})
```
CORS
```ts
cors({
  origin: "http://localhost:5173",
  credentials: true,
})
```
## 改善5: 定数化
```ts
const COOKIE_NAME = "session_id"
const SESSION_PREFIX = "session:"
const SESSION_TTL = 3600
```
## 改善6: デバッグログ強化
```ts
console.log({
  cookie: c.req.header("cookie"),
  sessionId,
  redisValue,
  ttl
})
```
# 学び

- Redis server と client は別物
- isReady が重要
- Cookie + Redis で初めてセッションが成立
- logout はセッション削除であり接続切断ではない
- Docker volume にデータは保存される

# まとめ

## 今回の実装は：
- Hono + Redis + Cookie によるセッション認証

## 問題の本質：
- Redis client 管理ミス
- セッション連携の不整合

## 解決方法：
- Redis client を1つに統一
- Cookie / fetch / Redis キーを揃える
- lifecycle を正しく管理する
